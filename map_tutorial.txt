## JSON-карты
Игра поддерживает подключаемые карты в формате JSON. Каждая карта хранится отдельным файлом в папке:
```text
maps/

Пример:

maps/classic_mansion.json
maps/mansion_crossroads.json

Название файла должно совпадать с id карты.

Например, файл:

classic_mansion.json

должен содержать:

{
  "id": "classic_mansion"
}

⸻

Общая структура карты

Пример структуры JSON-карты:

{
  "id": "classic_mansion",
  "title": "Классический особняк",
  "description": "Стабильная рабочая карта особняка.",
  "variant": 0,
  "board": {
    "w": 17,
    "h": 17
  },
  "paths": [
    [5, 4],
    [6, 4],
    [7, 4]
  ],
  "starts": {
    "Алекс Громов": [8, 9],
    "Мария Скарлет": [7, 9],
    "Профессор Фиолетов": [9, 9],
    "Виктор Олив": [8, 8],
    "Елена Белая": [7, 8],
    "София Синяя": [9, 8]
  },
  "rooms": {
    "Кухня": {
      "x1": 0,
      "y1": 0,
      "x2": 4,
      "y2": 4,
      "door": [4, 4],
      "secret": "Кабинет",
      "theme": "kitchen"
    }
  }
}

⸻

id

Уникальный идентификатор карты.

"id": "classic_mansion"

id должен совпадать с названием файла.

Правильно:

classic_mansion.json
"id": "classic_mansion"

Неправильно:

classic_mansion.json
"id": "another_map"

⸻

title

Название карты, которое отображается в лобби, игре и просмотрщике карт.

"title": "Классический особняк"

⸻

description

Краткое описание карты.

"description": "Стабильная рабочая карта особняка."

⸻

variant

Числовой вариант карты.

"variant": 0

Используется как дополнительный технический параметр для визуального различия и совместимости.

⸻

board

Размер игрового поля.

"board": {
  "w": 17,
  "h": 17
}

На текущем этапе рекомендуется использовать размер:

17 × 17

Поддержка других размеров заложена, но большие карты требуют отдельного тестирования интерфейса, баланса и производительности.

⸻

paths

Список клеток-коридоров, по которым игроки могут перемещаться.

"paths": [
  [5, 4],
  [6, 4],
  [7, 4]
]

Каждая точка задаётся координатами:

[x, y]

Правила для paths:

- клетки должны находиться внутри границ поля;
- клетки не должны находиться внутри комнат;
- все клетки должны быть связаны между собой;
- от стартовой зоны должна быть достижима каждая дверь комнаты;
- не должно быть изолированных островов коридора.

⸻

starts

Стартовые позиции персонажей на конкретной карте.

"starts": {
  "Алекс Громов": [8, 9],
  "Мария Скарлет": [7, 9],
  "Профессор Фиолетов": [9, 9],
  "Виктор Олив": [8, 8],
  "Елена Белая": [7, 8],
  "София Синяя": [9, 8]
}

Ключ — имя персонажа.
Значение — координата [x, y].

Правила для starts:

- стартовая позиция должна быть внутри поля;
- стартовая позиция не должна быть внутри комнаты;
- стартовая позиция должна находиться на клетке `paths`;
- стартовые позиции не должны дублироваться;
- все текущие персонажи должны быть указаны.

Сами персонажи пока задаются в коде проекта, а карта задаёт только их стартовые координаты.

⸻

rooms

Список комнат карты.

"rooms": {
  "Кухня": {
    "x1": 0,
    "y1": 0,
    "x2": 4,
    "y2": 4,
    "door": [4, 4],
    "secret": "Кабинет",
    "theme": "kitchen"
  }
}

На текущем этапе карта должна содержать все обязательные комнаты:

Кухня
Бальный зал
Оранжерея
Столовая
Бильярдная
Библиотека
Гостиная
Холл
Кабинет

Это связано с текущей карточной логикой игры.

⸻

Координаты комнаты

Комната задаётся прямоугольником:

"x1": 0,
"y1": 0,
"x2": 4,
"y2": 4

Где:

x1, y1 — верхний левый угол комнаты
x2, y2 — нижний правый угол комнаты

Правила:

- комната должна находиться внутри границ поля;
- координаты должны образовывать корректный прямоугольник;
- комнаты не должны пересекаться друг с другом.

⸻

door

Дверь комнаты.

"door": [4, 4]

Правила для двери:

- дверь должна находиться внутри комнаты;
- дверь должна находиться на границе комнаты;
- рядом с дверью должна быть клетка `paths`;
- соседняя клетка `paths` должна быть достижима от стартовой зоны.

⸻

secret

Секретный проход из комнаты.

"secret": "Кабинет"

Если секретного прохода нет:

"secret": null

Правила:

- `secret` должен вести в существующую комнату;
- для классической логики Cluedo секретные проходы желательно делать взаимными.

Пример взаимного прохода:

"Кухня": {
  "secret": "Кабинет"
},
"Кабинет": {
  "secret": "Кухня"
}

⸻

theme

Визуальная тема комнаты.

"theme": "kitchen"

Сейчас используется как задел под будущий визуал и карточки.

⸻

Как добавить новую карту

1. Создать новый JSON-файл в папке:

maps/

Пример:

maps/mansion_new.json

2. Указать внутри карты такой же id:

"id": "mansion_new"

3. Заполнить основные блоки:

title
description
board
paths
starts
rooms

4. Добавить карту в список доступных карт в available_maps().
5. Запустить валидатор:

php tools/validate_maps.php

6. Открыть preview:

/tools/map_preview.php

7. Проверить:

- статус карты OK;
- все комнаты отображаются;
- двери стоят корректно;
- paths связаны;
- секретные проходы отображаются;
- стартовые позиции находятся на коридорах;
- анализ расстояний не показывает явного перекоса.

8. Только после этого карту можно тестировать в реальной игре.

⸻

Валидатор карт

Для проверки карт используется файл:

tools/validate_maps.php

Запуск из терминала:

php tools/validate_maps.php

Валидатор проверяет:

- валидность JSON;
- совпадение id с именем файла;
- наличие title;
- board.w и board.h;
- наличие обязательных комнат;
- координаты комнат;
- выход комнат за границы поля;
- пересечение комнат;
- наличие дверей;
- расположение дверей на границах комнат;
- наличие соседней клетки коридора у двери;
- достижимость дверей от стартовой зоны;
- структуру paths;
- выход paths за границы поля;
- paths внутри комнат;
- связность paths;
- отсутствие изолированных участков коридора;
- secret-проходы;
- стартовые позиции персонажей.

⸻

Preview карт

Для визуальной проверки карт используется:

tools/map_preview.php

Preview показывает:

- игровую сетку;
- комнаты;
- двери;
- коридоры;
- стартовые позиции;
- secret-проходы;
- ошибки валидатора;
- режим достижимости;
- анализ расстояний;
- техническую информацию карты.

Preview нужен для разработки и проверки карт до запуска реальной партии.

⸻

Анализ расстояний

Preview содержит минимальный анализ расстояний от стартовых позиций до комнат.

Он показывает:

- минимальную дистанцию до комнаты;
- среднюю дистанцию до комнаты;
- максимальную дистанцию до комнаты;
- предупреждения о потенциальном перекосе.

Важно:

Валидатор отвечает на вопрос: "Карта технически рабочая?"
Анализ расстояний отвечает на вопрос: "Нет ли подозрительного перекоса?"

Жёлтое предупреждение в анализаторе не означает, что карта сломана. Оно означает, что карту стоит дополнительно проверить вручную.

⸻

Текущее ограничение

Сейчас разные карты могут менять:

- размер поля;
- расположение комнат;
- двери;
- коридоры;
- secret-проходы;
- стартовые позиции;
- визуальные темы.

Пока не рекомендуется менять:

- набор комнат;
- набор персонажей;
- набор оружия.

Причина: текущая карточная логика игры ещё использует фиксированные названия.

Будущий этап — переход на систему карточек с внутренними ID.
